# Artefact Skip Policy

## Rule

When a subagent returns `SKIP` or a JSON section is `null`, the skill's write decision depends on the **file type** — not a blanket "create everything" or "skip everything."

## Decision Table

| File type | Example | Create when SKIP? | Reason |
| ---- | ---- | ---- | ---- |
| **Scoped rule** (`paths:`-bearing `.md`) | `frontend-design-system.md`, `frontend-components.md` | **NO** | The file auto-loads into Claude's context on every matching file edit. A file that says "nothing detected" still activates — injecting false context into every component edit. Active harm. |
| **Mermaid diagram** (`.mmd`) | `frontend-data-flow.mmd`, `api-data-flow.mmd` | **NO** | An empty or stub Mermaid file fails `validate-mmd` and the hook will report errors. No diagram is better than an invalid one. |
| **JSON state file** | `component-registry.json`, `frontend-analysis.json`, `api-contracts-analysis.json` | **YES — write empty structure** | Downstream consumers (component-creator, other skills) always do a file-exists check + parse. An empty `[]` or `{}` with schema fields is far better than a missing file that causes a hard stop. |
| **Reference doc** (`.claude/docs/reference-*.md`) | `reference-component-inventory.md`, `reference-component-registry.md` | **YES — write placeholder** | Pure documentation. A "nothing detected" note does not harm Claude's behaviour and signals to the user that the area was scanned. |
| **Primary template** (`.claude/docs/*-creation-template.md`) | `component-creation-template.md` | **YES — write with "not observed" sections** | Always created. SKIP sections become a single line: `> Not observed in this project.` — never omit the section entirely. |

## Empty Structure Formats

When writing a JSON state file in SKIP case, use the minimal valid schema:

```json
// component-registry.json — empty
[]

// frontend-analysis.json — empty, schema present
{
  "schema_version": "1.1",
  "generated": { "ts": "<ISO-UTC>", "plugin_version": "<version>" },
  "frontend_roots": []
}
```

## What to Write in a Placeholder Reference Doc

```markdown
# <Title>

> Generated by `/<skill-name>` on <date>.
> No <area> detected in this project during analysis.
> Re-run `/analyze-<x>` if the codebase has changed.
```

No tables, no empty headings — just the notice. This keeps the file valid Markdown and useful as a signal.

## Report Requirement

Whenever a file is **skipped** (scoped rule or diagram not written because of SKIP), the skill's run report MUST note it explicitly:

```
Skipped: frontend-design-system-desktop.md — design-system-scanner returned SKIP (no token source detected)
```

This prevents silent gaps where the user doesn't know why a file is missing.

## Applies To

Every skill that writes target-project artefacts from subagent output:

- `create-frontend-docs`
- `update-frontend-docs`
- `create-api-contracts-docs`
- `update-api-contracts-docs`
- Any future `create-<x>-docs` / `update-<x>-docs`

Skills that write **toolkit-internal** files (e.g. `init-project` writing `.gitkeep` scaffolding) are exempt — they always create all directories.
